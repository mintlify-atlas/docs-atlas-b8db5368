## Prerequisites

Make sure you have the following installed before you begin:

| Tool | Version | Purpose |
|---|---|---|
| Java | 17+ | Run the Spring Boot backend |
| Maven | 3.9+ (or use the included `mvnw` wrapper) | Build the backend |
| Node.js | 18+ | Run the React frontend |
| PostgreSQL | 14+ | Primary database |

---

## Set up the backend

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Ericthered123/poo2025-Junin-grupo-Doyle-Olano-Uribe.git
    cd poo2025-Junin-grupo-Doyle-Olano-Uribe
    ```
  </Step>

  <Step title="Create and configure the environment file">
    Copy the example file and fill in your local values:

    ```bash
    cp .env.example .env
    ```

    Open `.env` and set the following variables:

    ```bash
    DB_URL=jdbc:postgresql://localhost:5432/TorneosGestion
    DB_USER=your_postgres_user
    DB_PASSWORD=your_postgres_password

    ADMIN_EMAIL=admin@example.com
    ADMIN_PASSWORD=your_admin_password

    JWT_SECRET=a_long_random_secret_string
    PASSWORD_PEPPER=another_random_secret
    ```

    <Note>
      Spring Boot does not read `.env` files automatically. Use the [EnvFile plugin](https://plugins.jetbrains.com/plugin/7861-envfile) in IntelliJ, or export each variable into your shell before running the app.
    </Note>
  </Step>

  <Step title="Create the PostgreSQL database">
    Connect to PostgreSQL and create the database:

    ```sql
    CREATE DATABASE "TorneosGestion";
    ```

    Spring Boot with JPA/Hibernate will create the schema automatically on first run.
  </Step>

  <Step title="Load environment variables and start the backend">
    If you are running from the command line (not IntelliJ), export the variables first:

    ```bash
    export DB_URL=jdbc:postgresql://localhost:5432/TorneosGestion
    export DB_USER=your_postgres_user
    export DB_PASSWORD=your_postgres_password
    export ADMIN_EMAIL=admin@example.com
    export ADMIN_PASSWORD=your_admin_password
    export JWT_SECRET=a_long_random_secret_string
    export PASSWORD_PEPPER=another_random_secret
    ```

    Then start the backend using the Maven wrapper:

    ```bash
    ./mvnw spring-boot:run
    ```

    On Windows:

    ```cmd
    mvnw.cmd spring-boot:run
    ```

    The backend starts on **port 8080**. You should see a log line like:

    ```
    Started TorneosApplication in X.XXX seconds
    ```
  </Step>
</Steps>

## Set up the frontend

<Steps>
  <Step title="Navigate to the frontend directory">
    ```bash
    cd frontend
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    The frontend starts on **port 5173** by default. Open [http://localhost:5173](http://localhost:5173) in your browser.
  </Step>
</Steps>

## Verify the setup

### Browse the home page

Open [http://localhost:5173](http://localhost:5173). You should see the public homepage listing any published tournaments. If no tournaments exist yet, the list will be empty — that is expected.

### Make your first API call

Fetch the list of published tournaments directly from the REST API:

```bash
curl http://localhost:8080/api/v1/tournaments
```

A successful response returns a JSON array (empty if no tournaments have been published yet):

```json
[]
```

### Register a participant account

```bash
curl -X POST http://localhost:8080/api/v1/account \
  -H "Content-Type: application/json" \
  -d '{
    "email": "participant@example.com",
    "password": "password123",
    "firstName": "Jane",
    "lastName": "Doe",
    "documentType": "DNI",
    "documentNumber": "12345678"
  }'
```

### Log in and get a JWT token

```bash
curl -X POST http://localhost:8080/api/v1/auth \
  -H "Content-Type: application/json" \
  -d '{"email": "participant@example.com", "password": "password123"}'
```

The response includes a `token` field. Pass it as a Bearer token in subsequent requests:

```bash
curl http://localhost:8080/api/v1/inscriptions \
  -H "Authorization: Bearer <your_token_here>"
```

## Explore the API with Swagger UI

The backend ships with an interactive Swagger UI. While the backend is running, open:

```
http://localhost:8080/swagger-ui/index.html
```

You can browse every endpoint, see request/response schemas, and execute calls directly from the browser.

<Tip>
  Use the **Authorize** button in Swagger UI to enter your JWT token once and authenticate all subsequent requests in the session.
</Tip>
